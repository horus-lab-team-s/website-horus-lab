C'est l'une des premières questions que l'on nous pose : « Faut-il commencer par un site web ou une application mobile ? » La bonne réponse dépend de vos utilisateurs et de vos objectifs.

## Commencez par vos utilisateurs

- **Vos clients sont surtout sur mobile, hors connexion stable ?** Une application mobile (ou une PWA) a du sens.
- **Vous visez la découverte et le référencement ?** Le web reste incontournable.
- **Budget serré et besoin de valider vite ?** Une **PWA** offre le meilleur des deux mondes.

## Le cas de la PWA

Une *Progressive Web App* est un site web qui s'installe et fonctionne comme une application : icône sur l'écran d'accueil, mode hors-ligne, notifications. C'est souvent le **point de départ idéal** dans nos contextes :

1. un seul code à maintenir ;
2. pas de friction d'installation via les stores ;
3. des coûts maîtrisés.

> Le natif (iOS/Android) reste pertinent quand vous avez besoin de performances poussées ou de fonctionnalités matérielles avancées.

## Notre recommandation

Validez d'abord votre idée avec une version simple et rapide à mettre sur le marché, puis investissez dans le natif une fois la traction prouvée. C'est l'approche **agile** que nous privilégions : livrer de la valeur tôt, puis itérer.
